tcpmon: An open-source utility to Monitor A TCP Connection

tcpmon is an open-source utility for monitoring the data flowing on a TCP connection. tcpmon is used by placing it in-between a client and a server. The client is made to connect to tcpmon, and tcpmon forwards the data to server along-with displaying it in its GUI.

Download: Three ways to launch tcpmon

  • Click here to run the latest version of tcpmon directly from Web. If a security warning dialog like the one shown here appears, click on the "Yes" button to grant tcpmon permissions to run.
  • Download tcpmon version 1.1 and simply double-click on the downloaded tcpmon.jar file to launch tcpmon.
  • Download tcpmon 1.1 plugin for the NetBeans IDE and monitor the traffic directly from the IDE.

Using tcpmon

The client will connect to the Local Port on the host where tcpmon is running. The address of the server should be specified in Server Name and Server Port. Information captured for each connection is displayed in a tabbed panel identified by the local port of the connection.

See Lukas Jungmann's blog for usage of tcpmon's NetBeans plugin.

Note that the client needs to connect to the machine where tcpmon is running on the "Local Port", not directly to the server.

tcpmon is authored by Inderjeet Singh and is hosted on the java.net community at http://tcpmon.dev.java.net/. tcpmon is a free software, and its source-code is available under the BSD License.

Supported Platforms (Java 1.4+): Windows XP/2000, MacOSX, Linux, Solaris

Supported NetBeans versions: NetBeans 5.5
